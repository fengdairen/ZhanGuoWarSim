# WarSim 战役模拟

WarSim 是一个命令行战役模拟器。战场宽度为 n，双方队伍数量可不同，按中轴线对齐后在同位进行 1v1 互砍，并在条件满足时触发夹击。

## 运行方式

### 编译

Windows (MinGW 示例):

```
chcp 936
```

```
g++ -std=c++17 -O2 -finput-charset=UTF-8 -fexec-charset=GBK main.cpp sim.cpp io.cpp battle.cpp -o warsim.exe
```

### 运行

```
./warsim.exe
```

## 核心概念

- **战场位置**: 位置 1..n 每回合同位 1v1 互砍；空位可被夹击跨越。
- **横向补位**: 回合开始时若本位置正面无敌军，优先向中轴线一侧补位（该侧有敌军）并移动 1 格；否则向背离中轴线一侧补位。补位按当前位置顺序逐个结算，后续单位会看到前序补位后的新位置，避免“双方错过”。
- **纵向补位**: 队伍数量可超过战场宽度 n，超出的单位按中轴线中心向外分配到各位置的“后排队列”。若某位置连续 2 回合没有我方前排单位，则该位置从后排入场 1 个单位（入场血量为初始血量的 80%），入场回合即可参战。
- **夹击触发**: 补位完成后本位置正面无敌军，且左右任一侧存在敌军，即可触发夹击。
- **夹击目标选择**:
  - 可跳过空位，优先选择距离最近的敌军。
  - 若距离相同，再按“更弱”目标选择（血量或士气）。
- **夹击伤害衰减**:
  - 距离每增加 1，伤害再乘一次倍率（最终倍率 = 夹击倍率^距离）。

### 回合流程摘要

1) 纵向补位：空位持续两回合后由后排补入。
2) 横向补位：按位置顺序处理，依据中轴线优先侧移动 1 格。
3) 同位 1v1 互砍并结算伤害。
4) 触发夹击并结算夹击伤害（按补位后状态判断）。

## 主要公式

- 基础伤害 = 攻击 * (1 + 训练度^0.5) * (满编率^0.8) * 战斗类型乘数
- 血量 = 士气 * (1 + 组织度/100) * (1 + 训练度^0.2) * (满编率^1.2)
- 士气 = 基础士气 * 乘数 - 时间损耗 - 战中损耗

## 交互说明

- 启动后可用一行快速输入全局参数：命中 k、战斗类型乘数、特殊情况乘数、最大回合数、战场宽度 n、A/B 队伍数量、夹击倍率；直接回车则逐项输入。
- 之后分别输入 Side A / Side B 的队伍参数（可一行快速输入，也可逐项输入）。
- 参数输入完毕后进入**主菜单**，可选择单次推演、百次测试、或修改参数。

### 单次推演模式
- 每回合输出简要战报（存活、阵亡、夹击统计）。
- 可输入多个位置编号（空格分隔）查看该回合的详细战报。
- 推演结束后可再次推演、运行百次测试、修改参数或退出。

### 百次测试模式
- 使用当前参数自动运行 100 场战斗（不输出中间战报）。
- 输出结果：
  - **A/B 胜率**与**平局率**
  - **A/B 平均残余兵力数**
- 适合用于验证兵种数值平衡性或批量对比不同参数下的胜负概率。

## 参数范围

- 比率类参数（训练度、满编率、护甲减伤率、防御率）使用 0-1。
- 组织度使用 0-100。
- 攻击、闪避、基础士气、时间损耗、战中损耗系数为非负实数。

## 输出说明

- **简要战报**: 显示双方存活数量、当回合阵亡位置、夹击次数与命中数，并用两组 0/1 可视化当前回合的阵列。
- **补位后位形**: 补位完成后、战斗结算前的阵列状态。
- **结算后位形**: 同位互砍与夹击结算后的阵列状态。
- **详细战报**: 输入位置编号后显示该位置 1v1 互砍与夹击记录（含夹击距离与最终倍率）。

## 代码结构

- `main.cpp`: 主入口、回合循环、`run_battle()` 无交互战斗引擎、`run_hundred_test()` 百次测试。
- `battle.cpp` / `battle.hpp`: 站位与横向补位逻辑、位形可视化。
- `sim.cpp` / `sim.hpp`: 公式与战斗数值计算。
- `io.cpp` / `io.hpp`: 输入与菜单交互。
- `types.hpp`: 数据结构定义。

## 开发备注

- 夹击触发基于“补位后的生存状态”，不会因本回合击杀而立刻触发夹击。
- 如需扩展为“每个队伍不同参数”，可将 `SideInputs` 与 `UnitState` 绑定为数组结构。
- 输出量较大时建议增加“简洁/详细模式”开关。
