# WarSim 战役模拟

WarSim 是一个命令行战役模拟器。战场宽度为 n，**每方可拥有多个不同属性的兵种**（每个兵种有独立的攻击、防御、士气等参数和队伍数），按中轴线对齐后在同位进行 1v1 互砍，并在条件满足时触发夹击。

## 运行方式

### 编译

Windows (MinGW):

```
g++ -std=c++17 -O2 -finput-charset=UTF-8 -fexec-charset=GBK main.cpp sim.cpp io.cpp battle.cpp -o warsim.exe
```

### 运行

```
warsim.exe
```

## 核心概念

### 多兵种系统

- 每方在 CSV 中可定义多个兵种（多行 `sideA`/`sideB`），每个兵种有独立的 10 项属性 + 队伍数。
- 启动时按**预期战斗力**（`基础伤害 × 初始血量`）降序排序。
- 展平为顺序列表后布阵：前排按中轴线居中放置，超出的兵力放入各位置的后排队列。

- **战场位置**: 位置 1..n 每回合同位 1v1 互砍；空位可被夹击跨越。
- **横向补位**: 回合开始时若本位置正面无敌军，优先向中轴线一侧补位（该侧有敌军）并移动 1 格；否则向背离中轴线一侧补位。补位按当前位置顺序逐个结算，后续单位会看到前序补位后的新位置，避免"双方错过"。
- **纵向补位**: 总兵力可超过战场宽度 n，超出单位按中轴线中心向外分配到各位置的"后排队列"。若某位置连续 `fill_delay` 回合没有我方前排单位，则从该位置的后排队列弹出 1 个单位入场（入场血量为初始血量的 80%），入场回合即可参战。
- **夹击触发**: 补位完成后本位置正面无敌军，且左右任一侧存在敌军，即可触发夹击。
- **夹击目标选择**:
  - 可跳过空位，优先选择距离最近的敌军。
  - 若距离相同，再按“更弱”目标选择（血量或士气）。
- **夹击伤害衰减**:
  - 距离每增加 1，伤害再乘一次倍率（最终倍率 = 夹击倍率^距离）。

### 回合流程摘要

1) 纵向补位：空位持续 `fill_delay` 回合后由后排补入。
2) 横向补位：按位置顺序处理，依据中轴线优先侧移动 1 格。
3) 同位 1v1 互砍并结算伤害。
4) 触发夹击并结算夹击伤害（按补位后状态判断）。

## 主要公式

- **基础伤害** = 攻击 × (1 + 训练度^0.5) × (满编率^0.8) × 战斗类型乘数
- **命中率** = 攻击 / (攻击 + 闪避 × 命中k)
- **血量** = 士气 × (1 + 组织度/100) × (1 + 训练度^0.2) × (满编率^1.2)
- **士气** = 基础士气 × 乘数 - 时间损耗 - 战中损耗
- **预期战斗力** = 基础伤害 × 初始血量（用于兵种排序）

## 交互说明

- 启动时自动从 `data.csv` 读取全部参数（全局参数 + 所有兵种），无需手动输入。
- 读取完毕后进入**主菜单**，可选择单次推演、百次测试、修改参数、重新读取 CSV 或退出。
- **程序内修改参数仅内存生效，不会写回 CSV 文件**。如需持久化修改，请直接编辑 `data.csv`，然后使用菜单中的**选项 4（重新读取 CSV）** 使之生效，或重启程序。

### 主菜单

| 选项 | 功能 |
|---|---|
| **1 单次推演** | 逐回合输出战报，可输入位置编号查看详细信息 |
| **2 百次测试** | 快速模拟 100 场，输出胜率与平均残余兵力 |
| **3 修改全局参数** | 调整命中 k、倍率、战场宽度、夹击目标等 |
| **4 重新读取 CSV** | 从 `data.csv` 重新加载，丢弃内存中所有修改 |
| **0 退出** | 结束程序 |

### 单次推演模式
- 每回合输出简要战报（存活、阵亡、夹击统计、位形可视化）。
- 可输入多个位置编号（空格分隔）查看该回合的详细战报（含兵种索引）。
- **推演结束后**输出每方每个兵种的战报卡：

```
=== Side A 兵种战报 ===
  A#1 (攻=25 闪=18 士气=75): 初始=4 阵亡=2 存活=2 造成伤害=452.3
=== Side B 兵种战报 ===
  B#1 (攻=20 闪=20 士气=55): 初始=20 阵亡=18 存活=2 造成伤害=310.7
```

### 百次测试模式
- 自动运行 100 场战斗（不输出中间战报）。
- 输出：A/B 胜率、平局率、平均残余兵力。

### 修改全局参数（选项 3）
- 运行时调整全部全局参数，即时生效。
- 修改仅内存生效，不会写回 CSV。

## 数据文件 (data.csv)

程序启动时自动读取同目录下的 `data.csv`。格式为横向表头 + 数据行：

```
参数名,命中k,战斗类型乘数,特殊情况乘数,最大回合数,战场宽度,夹击倍率,夹击目标,纵向补位延迟
global,10,1,1,3000,8,0.4,1,2
参数名,攻击,闪避,训练度,满编率,护甲减伤率,防御率,组织度,基础士气,时间损耗,战中损耗系数,队伍数
sideA,25,18,0.6,1,0.6,0.4,30,75,0,0,4
sideB,20,20,0,1,0,0,5,55,0,0,20
```

- **第1行**: 全局表头（程序跳过）
- **第2行**: 全局参数（8个数值）
- **第3行**: Side 表头（程序跳过）
- **第4行起**: 每行一个兵种，分类为 `sideA`/`sideB`，后跟 10 个属性 + 队伍数

**可随意追加 sideA/sideB 行来添加新兵种**。程序启动时自动按战斗力排序。

**程序内修改参数仅内存生效，不会写回 CSV 文件**。如需持久化修改，请直接编辑 `data.csv`，然后使用菜单中的**选项 4（重新读取 CSV）** 使之生效，或重启程序。

### 全局参数

| 参数 | 说明 |
|---|---|
| 命中k | 命中公式分母常数，越大命中率越低 |
| 战斗类型乘数 | 伤害倍率 |
| 特殊情况乘数 | 士气倍率 |
| 最大回合数 | 超过该回合数强制结束 |
| 战场宽度 | 一维战场位置数 n |
| 夹击倍率 | 夹击伤害基础衰减系数（每距离乘一次） |
| 夹击目标 | 1=血量优先，2=士气优先 |
| 纵向补位延迟 | 空位持续 N 回合后排补入，默认 2 |

### 兵种属性

| 属性 | 范围 | 说明 |
|---|---|---|
| 攻击 | ≥0 | 基础攻击力 |
| 闪避 | ≥0 | 闪避值，影响命中率 |
| 训练度 | 0-1 | 指数型收益（伤害、血量） |
| 满编率 | 0-1 | 编制完整度 |
| 护甲减伤率 | 0-1 | 防御触发时的基础减伤 |
| 防御率 | 0-1 | 触发防御判定的概率 |
| 组织度 | 0-100 | 线性加成血量 |
| 基础士气 | ≥0 | 士气起点 |
| 时间损耗 | ≥0 | 每回合士气自然衰减 |
| 战中损耗系数 | ≥0 | 每点承受伤害附加士气损耗 |
| 队伍数 | ≥1 | 该兵种有多少个单位 |

## 代码结构

| 文件 | 职责 |
|---|---|
| `types.hpp` | 数据结构：`SideInputs`, `Regiment`, `UnitState`, `RegimentReport`, `DeployResult` 等 |
| `sim.hpp / sim.cpp` | 公式计算：伤害、命中、防御、士气、血量、战斗力排序 |
| `battle.hpp / battle.cpp` | 布阵引擎、横向补位、位形可视化（`deploy_regiments`） |
| `io.hpp / io.cpp` | CSV 解析、菜单交互（`read_csv_params`, `edit_global_menu`） |
| `main.cpp` | 主循环、多兵种战斗引擎（`run_battle_multi`）、百次测试 |
| `data.csv` | 参数配置文件 |

## 开发备注

- **击杀归属**：采用累计伤害统计，不精确分配击杀。
- **布阵原则**：兵种按预期战斗力降序排列；前排居中，后排按中心向外分配队列。
- **夹击触发**基于"补位后的生存状态"，不会因本回合击杀而立刻触发夹击。
- 如需扩展，`SideInputs` 与 `Regiment` 已解耦为独立结构，便于动态增减兵种。
